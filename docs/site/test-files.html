Guide 02

Keep the DSL small. Push depth into runtime.

Tarn files stay YAML-first: requests, assertions, captures, retries, polling, cookies, and optional Lua, without turning into a custom programming language.

Minimal test

name: Health check
steps:
  - name: GET /health
    request:
      method: GET
      url: "{{ env.base_url }}/health"
    assert:
      status: 200
      body:
        "$.status": "ok"

Request surface

  • JSON body: request.body
  • Form encoding: request.form
  • GraphQL: request.graphql
  • Multipart: request.multipart
  • Custom methods: any valid HTTP token, not just a fixed enum

Assertions

Body assertions are keyed by JSONPath. Operator maps combine with AND.

assert:
  status: 201
  body:
    "$.id": { is_uuid: true, not_empty: true }
    "$.createdAt": { is_date: true }
    "$.tags": { type: array, length: 2, contains: "smoke" }
    "$": "plain text response"

Whole-body text and JSON mismatches emit unified diffs in human and JSON reporting.

Captures and transforms

capture:
  user_id: "$.id"
  final_url:
    url: true
  request_id:
    header: "x-request-id"
  session_cookie:
    cookie: "session"

# later
url: "{{ env.base_url }}/users/{{ capture.user_id }}"
header: "{{ capture.request_id | split('-') | first | to_string }}"

Capture sources include JSONPath, header, cookie, body regex, status, and final URL. Transforms include first, last, count, join, split, replace, to_int, and to_string.

Polling, retries, cookies

  • Polling: re-executes until assertions in poll.until pass.
  • Retries: step-level retry count for transient failures.
  • Cookies: automatic jar by default, step-level off switch, or named jars for multi-user sessions.

Schema support

# yaml-language-server: $schema=https://raw.githubusercontent.com/NazarKalytiuk/hive/main/schemas/v1/testfile.json

The VS Code extension wires this schema automatically. For full language intelligence in any LSP 3.17 client — VS Code, the Claude Code plugin, Neovim, Helix, Zed, or Emacs — install the standalone tarn-lsp server with cargo install tarn-lsp. Diagnostics, hover with inline JSONPath evaluation, nested completion, and quick-fix code actions all come from the same schema this comment references.
